당신은 시니어 프론트엔드 & UX 엔지니어입니다.
다음 요구사항을 모두 반영해서 Next.js(앱 라우터) + TypeScript + Tailwind CSS 기반의
"공공 AX·클라우드 MSP·AI 컨설팅 문의" 랜딩페이지 프로젝트를 생성해 주세요.

[목표]
- 공공기관 및 기업 담당자가 읽고 바로 "무료 30분 상담 신청"을 하고 싶어지는 랜딩페이지
- 신뢰도, 전문성, 전환율에 최적화된 구조
- 반응형 (모바일 ~ 데스크탑)

[기술 스택]
- Next.js 14 (App Router, TypeScript)
- Tailwind CSS
- 폰트는 시스템 폰트 사용 (별도 폰트 세팅 불필요)
- 페이지는 `/` 하나만 우선 구현
- `/api/contact` 엔드포인트를 생성해서 폼 데이터를 콘솔에 로그만 찍도록 구현 (실제 메일 연동은 나중에)

[프로젝트 구조]
- `app/layout.tsx` : 공통 레이아웃, 메타 태그, 기본 스타일
- `app/page.tsx` : 랜딩페이지 메인 컴포넌트
- `app/api/contact/route.ts` : 상담 폼 제출 API (POST)
- `components/` 폴더에 섹션별 컴포넌트 분리
  - `HeroSection.tsx`
  - `PainPointsSection.tsx`
  - `SolutionsSection.tsx`
  - `PackagesSection.tsx`
  - `ProcessSection.tsx`
  - `ProfileSection.tsx`
  - `FaqSection.tsx`
  - `ContactSection.tsx`
- `types/contact.ts` : 폼 데이터 타입 정의

[디자인 가이드]
- 전체 톤: 공공/엔터프라이즈 친화적인 클린한 B2B 느낌
- 배경: 기본은 흰색, 섹션 단위로 살짝 회색 배경 섞기
- 버튼: Tailwind utility만으로 구성 (예: `rounded-full px-6 py-3 text-sm font-semibold shadow`)
- 여백: 모바일에서도 요소 간 간격이 충분히 느껴지도록 `space-y-6` 이상
- 아이콘은 lucide-react 사용 가능 (선택)

[섹션 구성 – 컴포넌트 상세 요구사항]

1) <HeroSection />
- 큰 헤드라인: "공공기관 AX·클라우드·AI 도입, 실패 없이 시작할 수 있게 돕습니다."
- 서브 헤드라인: "공공 클라우드 MSP·AI 챗봇·업무자동화까지 기획·아키텍처·구축 전략을 한 번에 컨설팅합니다."
- 3개 정도의 핵심 포인트 (아이콘 + 텍스트)
  - 공공기관 프로젝트 경험
  - 클라우드 MSP + AI + 보안 통합 시야
  - 정부지원사업·RFP 대응
- 메인 CTA 버튼: "무료 30분 상담 신청하기"
- 서브 CTA 버튼: "제안서·자료 요청하기"
- 오른쪽에는 간단한 카드형 박스:
  - "지금 상담을 신청하시면, 현재 상황에 맞는 AX·클라우드·AI 도입 방향을 30분 안에 가이드합니다." 같은 문구

2) <PainPointsSection />
- 섹션 제목: "이런 고민이 있으신가요?"
- 카드/리스트 형태로 공공기관·기업 담당자의 페인포인트 4~6개 나열
- 예시는 다음 문구를 활용:
  - AI/챗봇 도입은 필요해 보이지만 어디서부터 시작해야 할지 모르겠다
  - 클라우드 전환이 필요한 건 아는데 보안·인증·심사 대응이 막막하다
  - 외주사마다 말이 달라 누구 말을 믿어야 할지 애매하다
  - 내부 결재·보고에 쓸 자료·전략 문서가 부족하다

3) <SolutionsSection />
- 섹션 제목: "우리가 제공하는 솔루션"
- 3~4개의 카드 컴포넌트로 구성:
  - 공공 AX/AI 도입 전략 컨설팅
  - 공공 클라우드 MSP·멀티 CSP 전략 수립
  - RAG 기반 AI 챗봇·업무자동화 설계
  - 정부지원사업·RFP 대응 자문
- 각 카드에는:
  - 짧은 설명
  - 담당자가 얻는 결과 (예: "내부 보고용 문서까지 한 번에 준비")

4) <PackagesSection />
- 섹션 제목: "컨설팅 패키지"
- 테이블 또는 카드 3개:
  - Lite 진단 / AX·AI 전략 / 클라우드 MSP·운영 패키지
- 항목: 대상, 주요 내용, 기간, 제공 형태 등

5) <ProcessSection />
- 섹션 제목: "진행 방식"
- 4단계 타임라인 UI
  - 1. 무료 상담 신청
  - 2. 요구사항 정리 & 현황 파악
  - 3. 제안 및 범위 협의
  - 4. 계약 및 프로젝트 수행

6) <ProfileSection />
- 섹션 제목: "왜 우리가 적합한 파트너인가요?"
- 대표자(컨설턴트)의 프로필 영역:
  - 공공기관·클라우드 MSP·AI 프로젝트 경험 요약 (더미 텍스트로 구성)
- 신뢰 메시지:
  - "공공기관 담당자의 입장에서, 어떻게 보고서와 사업을 만들어가야 하는지까지 같이 고민합니다." 문구 포함

7) <FaqSection />
- 자주 묻는 질문 4~6개
- 아코디언 형태로 구현 (간단한 state 사용)
- 예시 질문:
  - 예산이 정해지지 않은 상태에서도 상담 가능한가요?
  - PoC(파일럿)부터 진행해볼 수 있나요?
  - 내부 보고용 자료도 같이 만들어주시나요?
  - 기존 온프레미스 시스템과 병행 운영 전략도 자문 가능한가요?

8) <ContactSection />
- 섹션 제목: "무료 30분 상담을 신청하세요"
- 설명 문구: "현재 상황을 간단히 남겨주시면, 공공 AX·클라우드·AI 도입 방향을 함께 설계해드립니다."
- 폼 필드:
  - 기관/회사명 (required)
  - 부서/직함 (optional)
  - 성함 (required)
  - 이메일 (required)
  - 연락처 (required)
  - 관심 분야 (checkbox group)
    - 공공 AX/AI 전략
    - 클라우드 MSP/운영
    - AI 챗봇/업무자동화
    - 정부지원사업/RFP 대응
  - 현재 고민/상황 (textarea)
  - 상담 희망 방식 (select: 온라인 / 오프라인 / 이메일)
- 폼 제출 시:
  - `/api/contact`로 POST 요청
  - 성공 시 상단에 "상담 신청이 접수되었습니다. 1~2영업일 내에 메일로 연락드리겠습니다." 라는 알림 표시

[코딩 스타일 가이드]
- 함수형 컴포넌트 + FC 타입
- 가능한 한 `components/` 폴더를 잘게 나누어 재사용성 있게
- Tailwind className은 지나치게 복잡하지 않게 정리
- 텍스트는 일단 한국어로 하드코딩 (나중에 i18n 고려 가능)
